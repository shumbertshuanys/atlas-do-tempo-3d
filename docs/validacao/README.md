# docs/validacao/ — Kit de validação de demanda (Chat 3)

Material **operacional** para validar, com professores reais, se existe demanda pela
**função central** do Atlas ("o que acontecia no mundo neste momento?") — e decidir,
com evidência, se as camadas educacionais congeladas (Etapas 6–9) reabrem ou **não**.

Não é documento conceitual, etapa, playbook ou handoff. É kit de campo, pedido
explícito do dono (não fere R7). A execução das conversas é do dono, no mundo real;
este kit apenas prepara e padroniza para que o resultado vire **evidência** (PG1).

## Arquivos

| Arquivo | O que é |
|---|---|
| `kit-validacao-professores.md` | Entregáveis 1–4: roteiro de demo (10 min), guia de entrevista (~10 perguntas), critérios de sinal/invalidação, perfil-alvo + mensagem de abordagem |
| `ficha-professor-template.md` | Entregável 5: ficha de 1 página (duplicar por professor), consolidado das 5, e as entradas prontas para o `DECISOES.md` |
| *(a preencher)* `ficha-p1.md` … `ficha-p5.md` | Uma por conversa — a evidência bruta |

## Como usar (ordem)

1. **Selecionar 5 professores** pelos perfis do kit §4.1 — diversidade sobre conveniência.
2. **Abordar** com a mensagem curta do kit §4.2 (pede opinião crítica, não vende).
3. **Preparar a demo** (kit §1.0) e **escolher o caminho** (§1.2): A público (padrão,
   qualquer máquina) ou B curatorial só-pending (local, com token — nunca seeded).
4. **Conduzir:** Bloco 1 do guia (dor, antes da demo) → demo de 10 min → Bloco 2
   (reação/uso/disposição). Abertas primeiro, produto depois.
5. **Registrar** em até 24h numa cópia da ficha (`ficha-pN.md`), citando frases,
   separando fato de interpretação.
6. **Consolidar** as 5 fichas na tabela do template; aplicar os critérios do kit §3.4;
   colar o veredito no `DECISOES.md` (entrada (b) já esboçada).

## Decisão-chave já embutida (verificada no código do frame)

Na porta **pública** só circula `corpus ∩ approved`. As cenas cósmica/GOE/K-Pg são
densas; **1789 mostra só 2 itens franceses** — Inconfidência, tráfico e escravidão estão
`pending` na fila Tier 0, e os itens `seeded-demo` só aparecem na face curatorial. Isso
não é falha a esconder: é o **diferencial de governança** a demonstrar (kit §1.2, Caminho
B). Nunca ligar "demonstração (seeded)" numa conversa — fonte não validada não representa
o produto.
